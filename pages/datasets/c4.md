# C4

_最后更新：2026-04-14_

## 概述  
C4（Colossal Clean Crawled Corpus）是源自 Common Crawl 的高质量网页文本数据集，经去重、过滤低质页面、移除代码/乱码等清洗流程构建；在 Llama 预训练数据中占比 **15%**，是当前最大规模的公开预训练语料之一。

## 详细内容  
据《百面大模型》第 2.1 节，C4 的技术细节如下：

- **来源与构建**：取自 Common Crawl 网页快照，经论文 *“Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer”*（Raffel et al., 2020）提出的清洗流程：  
  - 移除含 “javascript”, “html”, “css” 等标签的行；  
  - 过滤低熵页面（如大量重复字符）、含过多数字/符号页面；  
  - 去重（基于 n-gram fingerprint）；  
  - 仅保留英文页面（C4-en 子集）；  
- **规模**：原始 C4-en 约 750GB 文本（约 36B tokens），Llama 使用其子采样版本；  
- **在 Llama 中的配比**：**15%** — 是所有数据源中占比最高的单一语料；  
- **价值与挑战**：  
  - 优势：覆盖现实世界语言（新闻、论坛、博客），提升模型对日常表达、俚语、新词的鲁棒性；  
  - 挑战：需强大清洗 pipeline（Common Crawl 原始数据质量参差，含大量广告、导航栏、爬虫陷阱）；  
- **对比 WebText**：WebText 通过 Reddit 链接点赞数（≥3）筛选优质网页，C4 则依赖自动化规则，二者策略互补。

C4 是 `trends/ai_reliability_engineering` 中“数据质量即模型可靠性基石”的典型例证。
