# The Bitter Lesson

*最后更新：2026-04-10*

## 概述

Rich Sutton 于 2019 年发表的短文。核心论点：70 年 AI 研究史表明，利用大规模计算的通用方法长期来看远胜于注入人类知识的专用方法。

## 详细内容

### 核心论点

计算成本随摩尔定律持续下降，可用算力指数级增长。短期内，将人类领域知识编码进系统确实有效且令研究者满足；但长期来看，这条路会触顶并阻碍进一步突破。真正持续有效的是能随算力扩展的通用方法。

### 四条历史规律

1. AI 研究者总是倾向于把知识内建到系统中
2. 短期内有效，且让研究者获得成就感
3. 长期来看触顶，甚至阻碍进步
4. 突破最终来自另一方向：用搜索（search）和学习（learning）来扩展计算

### 四个领域的证据


| 领域    | 人类知识方法        | 最终胜出的方法                        |
| ----- | ------------- | ------------------------------ |
| 计算机象棋 | 基于棋局结构的专家规则   | 深度搜索 + 专用硬件（1997 年击败 Kasparov） |
| 计算机围棋 | 利用棋局特征避免搜索    | 大规模搜索 + 自我对弈学习                 |
| 语音识别  | 音素、声道模型等语言学知识 | HMM → 深度学习（大数据 + 大算力）          |
| 计算机视觉 | 边缘检测、SIFT 特征  | 卷积神经网络                         |


### 两条教训

1. **通用方法的力量**：搜索和学习是唯二能随算力无限扩展的方法
2. **心智内容的复杂性**：不要试图简化空间、对象、对称性等概念并内建它们；应构建能自行发现这些复杂性的元方法（meta-methods）

> 我们想要的是能像我们一样去发现的 AI，而不是包含我们已发现之物的 AI。
